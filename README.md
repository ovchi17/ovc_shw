# Анализ разговорной речи на русском языке

Мобильное приложение для автоматизированной оценки качества устной речи.
Пользователь записывает голос или загружает готовое аудио, система распознаёт текст
с пословной временной разметкой и анализирует речь по пяти параметрам, выдавая
балл от 0 до 100 как по каждому параметру отдельно, так и итоговый.

## Возможности

- Запись с микрофона и загрузка готового аудио (mp3, wav, m4a, ogg)
- Транскрипция с пословной временной разметкой (точность около 20–30 мс)
- Комплексный анализ речи:
    - Слова-паразиты — частота, плотность, разнообразие, серии подряд
    - Паузы — доля времени фонации, средняя длина речевого такта, доля длинных пауз
    - Темп — общий темп речи, артикуляционный темп, стабильность (CV)
    - Лексика — MATTR, MTLD, доля топ-слов
    - Синтаксис — длина высказываний, глубина дерева, MDD, клаузы, доля сложных предложений
- Динамика прогресса и тепловая карта активности
- Сравнение двух записей по всем параметрам
- Персонализированные упражнения по слабым параметрам пользователя
- Светлая и тёмная тема, кастомные слова-паразиты
- Веб-админка для управления данными

## Архитектура

Клиент-серверное приложение.
Мобильное приложение на Flutter общается с серверной частью на FastAPI по HTTP/REST.
На сервере работает модуль обработки речи: ffmpeg готовит аудио, WhisperX делает
транскрипцию с временными метками, далее пять независимых модулей считают метрики.
Сборщик результатов агрегирует баллы во взвешенный итоговый балл и сохраняет в SQLite.

## REST API (основные эндпоинты)

### Авторизация
- POST   /api/v1/auth/register                  — регистрация
- POST   /api/v1/auth/login                     — вход, выдача JWT-токена

### Главный экран и профиль
- GET    /api/v1/user/me                        — краткий профиль текущего пользователя
- GET    /api/v1/dashboard                      — данные главного экрана (средние баллы, радар)
- GET    /api/v1/user/profile                   — расширенная статистика профиля
- GET    /api/v1/user/goals                     — цели на текущий месяц
- PATCH  /api/v1/user/me                        — обновление имени
- GET    /api/v1/user/fillers                   — список кастомных слов-паразитов
- POST   /api/v1/user/fillers/{word}            — добавить кастомный паразит
- DELETE /api/v1/user/fillers/{word}            — удалить кастомный паразит

### Записи и анализ
- POST   /api/v1/recordings/upload              — загрузка аудио и запуск анализа
- GET    /api/v1/recordings/status/{task_id}    — статус фоновой задачи анализа
- GET    /api/v1/recordings                     — список записей пользователя
- GET    /api/v1/recordings/{id}/audio          — стриминг аудиофайла
- GET    /api/v1/recordings/compare             — сравнение двух записей по всем параметрам
- GET    /api/v1/analysis/{id}                  — полный результат анализа по ID

### Прогресс и советы
- GET    /api/v1/progress/overview              — сводка прогресса за период
- GET    /api/v1/progress/dynamics              — точки для графика динамики
- GET    /api/v1/progress/activity              — тепловая карта активности за 30 дней
- GET    /api/v1/progress/parameter_dynamics    — динамика каждого из 5 параметров
- GET    /api/v1/tips                           — персонализированные советы и упражнения

## Запуск

### Backend

1. Установить зависимости:
   ```
   cd diploma
   pip install -r requirements.txt
   ```
2. Скачать модель spaCy для русского:
   ```
   python -m spacy download ru_core_news_sm
   ```
3. (опционально) Создать файл .env с настройками:
   ```
   SECRET_KEY=your-secret-key
   ```
4. Запустить сервер:
   ```
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

После запуска:
- API доступно по адресу http://localhost:8000
- Swagger-документация — http://localhost:8000/docs
- Админ-панель — http://localhost:8000/admin

### Frontend

1. Установить зависимости:
   ```
   cd diploma-analise-speach
   flutter pub get
   ```
2. Запустить на эмуляторе или устройстве:
   ```
   flutter run
   ```

## Алгоритм оценки

Каждый из пяти модулей возвращает балл от 0 до 100.
Итоговый балл записи — взвешенная сумма баллов модулей:

S = 0.30 * S_паразиты + 0.25 * S_паузы + 0.20 * S_темп + 0.15 * S_лексика + 0.10 * S_синтаксис

Внутри каждого модуля метрики нормируются в баллы через одну из трёх функций:
- Гауссова — оптимум в одной точке (например, длинные паузы, идеал 0%)
- Плато — оптимум в диапазоне (например, темп 115–150 слов/мин)
- Левосторонняя гауссова — "чем больше, тем лучше" (MATTR, MTLD)
